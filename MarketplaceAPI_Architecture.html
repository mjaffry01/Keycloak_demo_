HOW A REQUEST FLOWS (example)
----------------------------

1) User gets token from Keycloak:
   POST http://localhost:8081/realms/marketplace/protocol/openid-connect/token
   -> receives access_token (JWT) with roles: admin/seller/buyer

2) User calls API with token:
   GET https://localhost:7029/api/me
   Authorization: Bearer <JWT>

3) API validates token:
   - checks signature using Keycloak public key (from keycloak:8080)
   - checks issuer/audience
   - extracts roles from token -> maps to [Authorize(Roles=...)]

4) If authorized, API performs business logic:
   - reads/writes data in Postgres (postgres:5432)
   - returns JSON response to Swagger/Postman
