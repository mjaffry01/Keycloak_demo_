MarketplaceAPI — Windows Run Guide

Postgres + Keycloak + Docker • PowerShell friendly • with copy buttons
Tip: Replace <YOUR_GITHUB_REPO_URL> and <YOUR_REPO_FOLDER> below

0) Install Docker Desktop (Windows)

Open PowerShell as Administrator, then run:

PowerShell (Admin)
winget install -e --id Docker.DockerDesktop
After install: Restart Windows → open Docker Desktop → wait until it says “Docker is running”.
Verify
docker --version
docker compose version

1) Download the project from GitHub

Clone (recommended):

PowerShell
git clone <YOUR_GITHUB_REPO_URL>
cd <YOUR_REPO_FOLDER>

Or: Code → Download ZIP → Extract → open PowerShell inside the extracted folder.

2) Start the system (build + run)

Run from the folder containing docker-compose.yml:

PowerShell
docker compose up -d --build
This will start 3 containers: PostgreSQL, Keycloak, Marketplace API.

Verify containers:

PowerShell
docker ps

3) Open Swagger + Keycloak

Swagger (API)
https://localhost:7029/swagger
If you see a certificate warning: click Advanced → Proceed (normal for local HTTPS).
Keycloak
http://localhost:8081

4) Get a token (PowerShell)

Example: Admin token

PowerShell
$body = "grant_type=password&client_id=marketplace-api&username=test.admin&password=Pass@123"
Invoke-RestMethod `
  -Method Post `
  -Uri "http://localhost:8081/realms/marketplace/protocol/openid-connect/token" `
  -ContentType "application/x-www-form-urlencoded" `
  -Body $body

For other roles, change only username:

  • Seller: test.seller
  • Buyer: test.buyer

5) Use token in Swagger

  1. Open Swagger
  2. Click Authorize
  3. Paste:
Authorization Header
Bearer <ACCESS_TOKEN>

Stop / Reset

Stop containers:

PowerShell
docker compose down

Reset (delete DB data):

PowerShell
docker compose down -v

Troubleshooting (quick)

API logs:

PowerShell
docker logs marketplace-api --tail 120

Keycloak logs:

PowerShell
docker logs marketplace-keycloak --tail 120

Postgres logs:

PowerShell
docker logs marketplace-postgres --tail 120
